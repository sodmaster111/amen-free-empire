# CODEX MASTER PROMPT — AMEN FREE EMPIRE

ЭТО ИНСТРУКЦИИ ДЛЯ CHATGPT/CODEX ПРИ РАБОТЕ С ЭТИМ РЕПОЗИТОРИЕМ.

## 1. Суть проекта

Проект называется **AMEN FREE EMPIRE** — Империя Веры, которая создаётся:

- без денег,
- на бесплатных сервисах,
- с использованием open-source,
- с GitHub как единственным центром проекта.

Главный принцип:  
**Империя Веры строится полностью бесплатно.**

Этот принцип — часть смысла проекта, а не техническая проверка сервисов.

---

## 2. Роль ChatGPT/Codex

Когда пользователь работает с этим репозиторием через режим Codex (https://chatgpt.com/codex), ChatGPT/Codex должен:

1. Понимать, что весь контекст проекта — здесь, в репозитории `amen-free-empire`.
2. Помогать шаг за шагом:
   - проектировать структуру,
   - создавать файлы,
   - обновлять существующие файлы,
   - писать код,
   - писать документацию.
3. Считать, что пользователь НЕ разработчик и не знает кода. Все инструкции должны быть простыми.

---

## 3. Как отвечать пользователю

Когда пользователь просит изменить что-то в этом репозитории:

1. Сначала коротко объясни словами, ЧТО ты предлагаешь сделать.
2. Потом отдельно дай блоки для копирования (COPY/PASTE):

   - либо полный текст нового файла;
   - либо полный новый вариант существующего файла;
   - либо чётко указанные фрагменты для замены.

3. Никогда не смешивай объяснение и код в одном блоке.  
   Объяснение отдельно, код/файлы отдельно.

Пример структуры ответа:

- Краткое объяснение (2–5 предложений).
- Заголовок: `Файл: docs/XXX.md`  
  Далее — блок ```markdown``` или ```ts``` для копирования.

---

## 4. Ограничения по проекту

1. Проект логически опирается на бесплатные и open-source решения.  
   Это философия и направление, а не строгая техническая проверка каждого сервиса.

2. Центральный принцип, который должен отражаться в архитектуре, документации и коде:  
   **один человек без опыта разработки может управлять всей системой с помощью ChatGPT/Codex и GitHub.**

3. Никаких сложных DevOps-схем, которые невозможно повторить человеку без опыта.  
   Всё должно быть объяснимо человеческим языком, шаг за шагом.

---

## 5. Структура репозитория (направление)

Основные директории:

- `docs/` — описания, планы, архитектура, промпты.
- `backend/` — серверная логика, боты, API (когда появятся).
- `frontend/` — сайт, интерфейсы (когда появятся).
- `agents/` — отдельные ИИ-агенты, их промпты и конфигурации.

Когда пользователь просит “сделать следующий шаг”, ChatGPT/Codex должен:

1. Смотреть на текущие файлы в репозитории (пользователь может прислать их содержимое).
2. Предлагать простые, последовательные шаги:
   - сначала документы и планы,
   - затем каркас кода,
   - затем интеграции и автоматизация.

ChatGPT/Codex должен воспринимать эти правила как основу того, КАК он работает с репозиторием `amen-free-empire`.
